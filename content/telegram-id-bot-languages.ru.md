@print_id_bot отвечает на **16 языках** в зависимости от настроек языка в Telegram. Ручной выбор языка не нужен.

## Как это работает

Бот читает ваш **language_code** из апдейта (из `message.from.language_code`, `edited_message.from.language_code` или `callback_query.from.language_code`). Использует первую часть кода (например `pt-BR` → `pt`) и сопоставляет с поддерживаемой локалью. Неподдерживаемые коды откатываются на **английский**.

## Поддерживаемые языки (16)

| Код | Язык |
|-----|------|
| `en` | Английский |
| `ru` | Русский |
| `fa` | Персидский |
| `uz` | Узбекский |
| `hi` | Хинди |
| `zh` | Китайский |
| `tr` | Турецкий |
| `pt` | Португальский |
| `es` | Испанский |
| `ar` | Арабский |
| `id` | Индонезийский |
| `de` | Немецкий |
| `ur` | Урду |
| `fr` | Французский |
| `tl` | Тагальский (филиппинский; `fil` сопоставляется с `tl`) |
| `am` | Амхарский |

## Особые случаи

- **Филиппинский** — Telegram отправляет `fil`; бот сопоставляет с `tl` (тагальский).
- **Португальский (Бразилия)** — `pt-BR` использует `pt` (португальский).
- **Китайский** — `zh-CN`, `zh-TW` используют `zh` (китайский).

## Что локализовано

Все ответы бота: приветственный текст, подписи (User ID, Chat ID, название группы), текст помощи, кнопки и сообщения вроде «ID hidden» или «Нажмите на число, чтобы скопировать». Сырой JSON в /json и /dump не меняется; только человекочитаемые подписи используют ваш язык.

## FAQ

### Как изменить язык бота?
Измените язык приложения Telegram в Настройки → Язык. Бот использует его. Переключателя языка в боте нет.

### Что если мой язык не поддерживается?
Бот откатывается на английский. Вся функциональность работает так же.

### Поддерживает ли бот языки с письмом справа налево (RTL)?
Да. Арабский, персидский и урду поддерживаются. Telegram обрабатывает отображение RTL.

### Могу ли я запросить новый язык?
Бот поддерживает 16 языков. Добавление новых зависит от сопровождающих. Откат на английский позволяет всем пользоваться ботом.

### Сохраняется или передаётся ли язык куда-то ещё?
Нет. Бот читает `language_code` из каждого апдейта и использует его только для выбора строк ответа. Он не сохраняется и не передаётся.
